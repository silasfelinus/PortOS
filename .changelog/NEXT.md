# Next Release

## Fixed

- **[issue-1459] Editorial LLM checks now guarantee a manuscript budget floor, so a large reverse outline can't starve the manuscript chunk to empty.** The manuscript-consuming editorial checks re-send a fixed context block (the reverse-outline scene map, character arcs, plotline coverage, style-guide expectations, …) on every chunk, counted as `overheadTokens`. On a provider with a small or fallback context window **and** a large reverse outline (many scenes → a large scene map), that overhead could meet or exceed the usable input budget — `planManuscriptPass` then returned `usableChars = 0` and the chunker sliced every manuscript chunk to `''`, so a default-enabled check silently ran on **no prose**. A check now declares its trimmable context as a `context: { varName }` map (instead of baking it into `overheadTokens`); the runner reserves a minimum manuscript budget (`MANUSCRIPT_FLOOR_TOKENS`) and, when the context would push the manuscript below that floor, **trims the context block** (largest first, so the unbounded scene map absorbs the cut while bounded blocks survive) and feeds the model the same trimmed block it was budgeted for — rather than overflowing the window or reviewing an empty manuscript. The floor decision lives centrally in `contextBudget.js` (`fitContextToManuscriptFloor`/`capContextOverhead`/`trimContextToBudget`), so every manuscript+context check benefits. Unit-tested end-to-end: a tiny window + a 400-scene outline still yields a non-empty manuscript chunk. (`server/lib/contextBudget.js`, `server/services/pipeline/editorial/checkRunner.js`, `server/lib/editorial/checkRegistry.js`, #1459)
