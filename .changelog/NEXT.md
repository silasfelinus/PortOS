# Unreleased Changes

## Added

- **[issue-730] Turn cross-machine resume on per session, right in the Story Builder** — each Story Builder session now has a "Cross-machine resume" toggle in its header, so you can opt a single session into syncing across your machines without leaving the wizard. Once it's on, a Reconcile button appears that adopts this machine's current universe/series records as the session's new staleness baseline — and it lights up only when those records have actually drifted from what the session last synced, so you know when reconciling will change something. Sessions stay private to one machine by default.
- **[issue-730] Story Builder sessions can sync across your machines** — a Story Builder session you mark for cross-machine resume now travels to your sync peers, so you can pick up the same wizard (its step locks, progress, and provider choice) on another machine. Sessions stay private to one machine by default; only the ones you explicitly opt in are shared, via a new "Story Builder" sync category you enable per peer on the Instances page. Local-only sessions are never pushed.
- **[issue-735] AudioLDM2 long-form music backend** — the pipeline audio stage can now generate background music with a second local engine, AudioLDM2, alongside MusicGen. Pick the engine in the Generate panel: MusicGen for short clips (up to 30s) or AudioLDM2 for longer pieces (up to two minutes), each with its own model choices. Both run entirely on your machine with no API key; install AudioLDM2's runtime with `INSTALL_AUDIOLDM2=1 bash scripts/setup-image-video.sh` and the panel shows an install hint until it's ready.
- **Shell arrow-key hot buttons** — the Shell page's quick-command toolbar now has touch-friendly Up / Down / Left / Right / Enter buttons that send the matching key to the terminal. Handy on phones and tablets where the on-screen keyboard hides the arrow keys, for scrolling command history, and for driving TUI apps without a hardware keyboard.
- **[issue-827] Clinician export for blood and lifestyle data** — MeatSpace gains an Export tab that builds a one-page, doctor-ready summary of your blood panels and lifestyle. Each blood test is laid out as a clean table with reference ranges and out-of-range markers flagged, and your lifestyle factors (sex, smoking, exercise, sleep, diet, stress, BMI, and any chronic conditions) are summarized above them. Print it or save it as a PDF straight from the browser, or copy the whole thing as markdown to paste into a message or note. Genome and longevity context are intentionally left out of this view.
- **Optional login password** — a new Settings → Security tab lets you set a single shared password that gates the PortOS UI, API, asset routes, and Socket.IO behind a sign-in screen. Off by default, so nothing changes if you don't enable it. Useful when other people, devices, or sidecars share your tailnet. Sessions last 30 days, persist across server restarts, and can be revoked by changing or clearing the password.
- **[issue-781] Writers Room live director** — turn on "Live Director" for a work and, whenever you pause while writing, the Creative Director offers a few short ways to continue from your cursor: a beat to take, a ready-to-insert line of prose, or a bit of dialogue. It's off by default and opt-in per work, waits until you stop typing before asking, and respects a daily suggestion budget so it never runs away with your AI usage.
- **[issue-841] Live render previews in the Writers Room** — with Live Director on, a "Render scene" button now renders a quick reference image for the scene your cursor sits in, shown in the same render dock as your storyboard renders and attached back to the scene. It has its own daily render budget, separate from the text-suggestion budget, since renders cost more.
- **[issue-710] Health insight provenance chips** — longevity and health views now label how each number was derived, the way Ask tags its sources. A small chip on your death-clock estimate, LEV tracker, epigenetic age, chronological age, and genome markers marks it as data-backed, inferred, experimental, or speculative — tap it for a plain-language explanation of how it was figured and what would change it. So a modeled projection never reads as a measured fact.
- **[issue-709] Review Hub "Needs Attention" queue** — the Review Hub now opens with a cross-domain queue that pulls everything currently waiting on you into one list: Brain inbox items that need classifying, Ask answers ready to promote, CoS tasks awaiting approval, unsent message drafts, critical/high health anomalies, and failed backups. The most urgent items sort to the top, each row opens straight to where you handle it, and you can dismiss anything you don't want to act on right now. A slow or unavailable source is noted instead of blocking the rest.
- **[issue-709] Review Hub queue refreshes itself** — the "Needs Attention" queue now updates the moment you handle something anywhere: resolve a CoS task, send or approve a message draft, or classify an inbox item on its own page and the matching row disappears (or a new one appears) without waiting for a reload. A failed or finished backup updates its row the same way. Anything you've dismissed this session stays dismissed.
- **[issue-709] Act on Review Hub items in place** — rows in the "Needs Attention" queue now carry an inline action so you can clear them without leaving the page: mark a Brain inbox item done, or approve a waiting CoS task or message draft. The row drops away the moment it's handled. Items that need a choice or have no one-click resolution (promoting an Ask answer to a note/task/goal, health anomalies that clear when the condition does, a failed backup retried from Backup settings) keep their open-and-dismiss controls.
- **[issue-709] Richer triage context in the Review Hub queue** — "Needs Attention" rows now show at-a-glance context so you can decide without opening them: a HIGH/MEDIUM/LOW badge on CoS approvals, the turn count on Ask answers, the recipient and channel on message drafts, where a Brain inbox item was captured from, and the category of a health anomaly. Each chip only appears when that detail exists, so nothing is invented to fill space.
- **[issue-709] Promote an Ask answer straight from the Review Hub** — Ask rows in the "Needs Attention" queue now offer inline "→ Brain" and "→ Task" buttons that save the conversation's latest answer as a Brain note or a CoS task in one click, then clear the row. Promoting to a Goal still opens the conversation, since it needs you to pick which goal.
- **[issue-709] Promote an Ask answer into a goal from the Review Hub** — Ask rows now also carry a "→ Goal…" picker: choose one of your active goals and the conversation's latest answer is logged as a progress entry on that goal, then the row clears — no need to open the conversation. The picker only appears when you have an active goal to promote into.
- **[issue-708] Digital Twin personas** — give your twin named contexts — Professional, Casual, Family, or your own — each with instructions for how it should modulate voice and behavior. Activate one from the new Personas tab and your CoS agents adopt that context when they act on your behalf, while building your base identity stays unaffected. Create, edit, switch, and clear personas at any time.

Ships a starter dilemma suite you can customize.
- **[issue-708] Test your twin as a persona** — the Digital Twin's Test tab can now run both the behavioral and values-alignment suites *as* a chosen persona instead of only the base twin: pick "Professional", "Casual", or any persona you've created and the suite embodies that context's instructions before answering. Each run is tagged in the history with the persona it used, so you can compare how the same model behaves across contexts. With no personas defined, the picker stays hidden and nothing changes.
- **[issue-708] Digital Twin adversarial boundary tests** — the Test tab gains a third mode that actively tries to *manipulate* your twin into crossing a line: each scenario applies a pressure tactic — authority pressure, flattery, guilt-tripping, incremental escalation, or reframing harm as help — and grades whether your twin held its boundary, wavered, or breached it. Per model you can see where it stays firm under pressure and where it folds. Runs as the base twin or any persona, tags each run with its score, and ships a starter scenario suite you can customize to the tactics that matter to you.
- **[issue-708] Digital Twin multi-turn conversation tests** — the Test tab gains a fourth mode that grades your twin across a whole conversation instead of a single reply: each scenario plays out its messages in order — the twin sees its own earlier answers as the thread continues — and scores whether it stayed consistent, contradicted itself, caved to repeated pushback, or forgot a constraint it agreed to a turn ago. Per model you can read the full transcript and see where consistency held or broke. Runs as the base twin or any persona, and ships a starter conversation suite you can customize.

Click a passage to highlight the scenes and media it became; click a scene to jump back to its source prose and see the image, prompt, and render details. - **[issue-717] No stray React warnings when you navigate away mid-action** — buttons that run an async task (save, generate, sync) no longer log an "update on an unmounted component" warning if you leave the page before the task finishes.
- **[issue-719] Steadier media galleries during live note/star sync** — incoming annotation broadcasts that match what a view already shows no longer trigger a needless re-render of the media cards.
- **[issue-724] Local video renders no longer hang after finishing** — if a local video model finishes writing its file but its process refuses to exit (a known teardown stall on some Apple-silicon runtimes), PortOS now force-stops it after a short grace period instead of leaving the render — and the next one in line — wedged indefinitely.
- **[issue-766] Episode Video renders cleanly when a remembered model is gone** — if the video model you previously pinned for an episode has since been removed, the render now quietly falls back to your default model instead of failing partway through.
- **[issue-728] Imported stories no longer fake a "Ready" aesthetic or reader map** — when you start a Story Builder session from an import, only the steps the importer actually fills (idea, plot arc, characters, issues) open as "Ready" to review. The Universe Aesthetic and Reader Map steps now start "pending" instead of showing an empty step under a misleading "Ready" badge, so you generate them like you would in a from-scratch build.
- **Self-update no longer crashes at the "Rebuilding esbuild" step** — the updater rebuilt esbuild via a hardcoded `node_modules/esbuild/install.js` path that assumed esbuild was hoisted to the top of each dependency tree. It no longer is (esbuild nests under Vite/Vitest, and the server dropped its esbuild dependency entirely with Vitest 4 → Vite 8), so the path 404'd and aborted the whole update with exit code 1. The updater now uses `npm rebuild esbuild`, which finds esbuild wherever it lives and skips cleanly when absent; the same fix lands in `npm run install:all` and the Windows updater.
- **[issue-796] Agent CPU/memory readouts work on Windows** — on Windows installs, the live CPU and memory stats for running agents always read as zero. The stats reader expected Unix `ps` output but was being handed Windows `tasklist`'s comma-quoted format, so it never found the numbers. It now reads the Windows format correctly, reporting real memory usage and marking a finished agent as done rather than leaving it in an unknown state.
- **[issue-731] Locked Story Builder steps flag stale when their own content is edited elsewhere** — locking a step (e.g. Universe Aesthetic) only gates the Story Builder wizard; the underlying universe and series records stay editable in Universe Builder, Arc Planner, and the cast editor. Editing a locked step's own content out-of-band — changing the universe logline, rewriting the arc, or adding a character — now marks that step stale so you re-review and re-lock it, instead of silently trusting a lock whose content has since changed.
- **[issue-851] An import-draft universe starts syncing once real work lands in it** — if you began an import (which creates a hidden draft universe) but attached a real, committed series to it without finishing the import, the universe could stay invisible to your other machines forever. Linking committed work to it now promotes it to a normal, syncing universe automatically — so it reaches your sync peers like any other.
