# Unreleased Changes

## Added

- **[issue-709] Review Hub "Needs Attention" queue** — the Review Hub now opens with a cross-domain queue that pulls everything currently waiting on you into one list: Brain inbox items that need classifying, Ask answers ready to promote, CoS tasks awaiting approval, unsent message drafts, critical/high health anomalies, and failed backups. The most urgent items sort to the top, each row opens straight to where you handle it, and you can dismiss anything you don't want to act on right now. A slow or unavailable source is noted instead of blocking the rest.
- **[issue-709] Act on Review Hub items in place** — rows in the "Needs Attention" queue now carry an inline action so you can clear them without leaving the page: mark a Brain inbox item done, or approve a waiting CoS task or message draft. The row drops away the moment it's handled. Items that need a choice or have no one-click resolution (promoting an Ask answer to a note/task/goal, health anomalies that clear when the condition does, a failed backup retried from Backup settings) keep their open-and-dismiss controls.
- **[issue-708] Digital Twin personas** — give your twin named contexts — Professional, Casual, Family, or your own — each with instructions for how it should modulate voice and behavior. Activate one from the new Personas tab and your CoS agents adopt that context when they act on your behalf, while building your base identity stays unaffected. Create, edit, switch, and clear personas at any time.
- **[issue-708] Digital Twin values-alignment tests** — the Digital Twin's Test tab gains a second test mode: instead of only checking whether your twin *sounds* like you, it poses ethical dilemmas and grades whether the twin's choice is consistent with your ranked values. Each dilemma names the values at stake and ships an "aligned" and "misaligned" reference, and runs score against the values hierarchy from your Identity tab — so you can see, per model, where your twin holds your values and where it drifts. Ships a starter dilemma suite you can customize.
- **[issue-708] Test your twin as a persona** — the Digital Twin's Test tab can now run both the behavioral and values-alignment suites *as* a chosen persona instead of only the base twin: pick "Professional", "Casual", or any persona you've created and the suite embodies that context's instructions before answering. Each run is tagged in the history with the persona it used, so you can compare how the same model behaves across contexts. With no personas defined, the picker stays hidden and nothing changes.
- **[issue-708] Digital Twin adversarial boundary tests** — the Test tab gains a third mode that actively tries to *manipulate* your twin into crossing a line: each scenario applies a pressure tactic — authority pressure, flattery, guilt-tripping, incremental escalation, or reframing harm as help — and grades whether your twin held its boundary, wavered, or breached it. Per model you can see where it stays firm under pressure and where it folds. Runs as the base twin or any persona, tags each run with its score, and ships a starter scenario suite you can customize to the tactics that matter to you.
- **[issue-708] Digital Twin multi-turn conversation tests** — the Test tab gains a fourth mode that grades your twin across a whole conversation instead of a single reply: each scenario plays out its messages in order — the twin sees its own earlier answers as the thread continues — and scores whether it stayed consistent, contradicted itself, caved to repeated pushback, or forgot a constraint it agreed to a turn ago. Per model you can read the full transcript and see where consistency held or broke. Runs as the base twin or any persona, and ships a starter conversation suite you can customize.
- **[issue-704] Writers Room synced review** — a new Review view shows a work's prose, its generated script scenes, and the images rendered from them side by side, with the links between them. Click a passage to highlight the scenes and media it became; click a scene to jump back to its source prose and see the image, prompt, and render details. Each media card carries its provenance, and the script is flagged as stale when the draft has changed since it was generated. Pick which of the three panes to show.
- **`/claim --issues`** — the claim workflow can now pull its work queue from GitHub issues instead of PLAN.md, auto-picking the oldest open issue filed by the repo owner and filing any major code-review findings as new issues. Developer tooling only — no app-facing change.
- **Episode video model picker** — the pipeline's Episode Video stage now lets you choose which video model renders each scene, alongside the aspect-ratio and quality controls. Leave it on "Default model" to follow your video settings, or pin a specific model; the choice is remembered when you restart a render.

When you supply references it loads the `FLUX.2-klein-9B-kv` repo (tuned for reference editing) instead of the base 9B; plain text and image-to-image renders stay on the base repo. ## Fixed

- **[issue-729] Style-probe renders stay put across federated machines** — a universe's base style-probe images are now kept local to each machine, so syncing with an older peer no longer wipes them and a probe render no longer surfaces a phantom sync conflict.
- **[issue-717] No stray React warnings when you navigate away mid-action** — buttons that run an async task (save, generate, sync) no longer log an "update on an unmounted component" warning if you leave the page before the task finishes.
- **[issue-719] Steadier media galleries during live note/star sync** — incoming annotation broadcasts that match what a view already shows no longer trigger a needless re-render of the media cards.
- **[issue-724] Local video renders no longer hang after finishing** — if a local video model finishes writing its file but its process refuses to exit (a known teardown stall on some Apple-silicon runtimes), PortOS now force-stops it after a short grace period instead of leaving the render — and the next one in line — wedged indefinitely.
- **[issue-766] Episode Video renders cleanly when a remembered model is gone** — if the video model you previously pinned for an episode has since been removed, the render now quietly falls back to your default model instead of failing partway through.
- **[issue-728] Imported stories no longer fake a "Ready" aesthetic or reader map** — when you start a Story Builder session from an import, only the steps the importer actually fills (idea, plot arc, characters, issues) open as "Ready" to review. The Universe Aesthetic and Reader Map steps now start "pending" instead of showing an empty step under a misleading "Ready" badge, so you generate them like you would in a from-scratch build.
- **Self-update no longer crashes at the "Rebuilding esbuild" step** — the updater rebuilt esbuild via a hardcoded `node_modules/esbuild/install.js` path that assumed esbuild was hoisted to the top of each dependency tree. It no longer is (esbuild nests under Vite/Vitest, and the server dropped its esbuild dependency entirely with Vitest 4 → Vite 8), so the path 404'd and aborted the whole update with exit code 1. The updater now uses `npm rebuild esbuild`, which finds esbuild wherever it lives and skips cleanly when absent; the same fix lands in `npm run install:all` and the Windows updater.
- **[issue-796] Agent CPU/memory readouts work on Windows** — on Windows installs, the live CPU and memory stats for running agents always read as zero. The stats reader expected Unix `ps` output but was being handed Windows `tasklist`'s comma-quoted format, so it never found the numbers. It now reads the Windows format correctly, reporting real memory usage and marking a finished agent as done rather than leaving it in an unknown state.
