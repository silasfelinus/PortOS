# Unreleased Changes

## Added

- **Manuscript editor: triage editorial feedback without the scroll-marathon** — clicking an editorial note used to scroll the page all the way to the bottom (the suggested-fix card rendered below the entire section), and getting to the next note meant scrolling all the way back up. Now the open note's card is injected **into the prose right at its anchor**, so the text being discussed and the Accept/Dismiss actions are on screen together; the card also gains a **‹ N of M › stepper** that walks every open note in story/anchor order (cross-issue, in both Live and Review modes), and **resolving a note auto-advances to the next open one** — working through 29 notes is one action per note, no sidebar round-trips. The sidebar index keeps its active row scrolled into view so it tracks where the pass is. (`client/src/components/pipeline/manuscript/`, `client/src/pages/PipelineManuscriptEditor.jsx`)

- **Manuscript impact preview shows a real diff instead of a red/green wall** — the whole-section before/after in **Impact preview** ran a flat word-level diff that tripped its safety cap on manuscript-sized sections and fell back to "everything red on the left, everything green on the right." Sections are now diffed with a new hunked renderer (`HunkDiff`): unchanged text collapses to "⋯ N unchanged lines ⋯" (click to expand, two context lines kept around each change) and each changed region gets precise word-level red/green highlights, so what an accepted batch of edits *actually changes* is legible at a glance. And the preview is no longer look-but-don't-touch: an **Accept all N edits** button applies every previewed note right from the modal (one accept per note, sequentially, through the same server route the cards use — each section keeps its version-history snapshot, so it's revertible). Notes the server rejects (moved anchor, overlapping edits) are reported and stay in the preview for a regenerate. The underlying word diff also trims common prefix/suffix text before hitting its cap, so the per-note edit diffs stay granular on long quotes too. (`client/src/lib/diffLines.js`, `client/src/components/ui/HunkDiff.jsx`, `client/src/lib/diffWords.js`, `client/src/components/pipeline/manuscript/ManuscriptImpactPreview.jsx`)

- **Editorial review can draft every fix in one pass** — the manuscript editor's **Run editorial review** now has a second checkbox, **Generate edits for every finding**, next to *Start fresh*. With it on, the completeness pass returns not just the itemized editorial notes but a concrete rewrite (`replace`) for each one, so every comment lands with its fix already attached — you go straight to reviewing the full diff and accepting edits one-by-one, with no per-note **Generate fix** click. The model returns each finding's edit inline (a single LLM pass, no N+1 round-trips), and the run streams per-chunk progress over SSE (`Drafting edits — chunk 2 of 4…`) with a Cancel button, re-attaching if you reload mid-run. Findings whose anchor can't be located verbatim stay advice-only and fall back to manual fix generation, so nothing presents an unappliable diff. Leaving the box off keeps today's findings-only behavior unchanged. The new `replace` field rides in a `{{#withEdits}}` block of the completeness prompt (migration 083; the prior prompt auto-upgrades on existing installs). (`server/services/pipeline/arcPlanner.js`, `manuscriptReview.js`, `manuscriptCompletenessRunner.js`, `client/src/pages/PipelineManuscriptEditor.jsx`)

- **Animated 3D mini-character CoS avatars (bundled, CC0)** — two new Chief-of-Staff avatar styles, **Mini Character — Male (3D)** and **Mini Character — Female (3D)**, ship as rigged, animated GLB models out of the box (no user setup, no download). Unlike the `muse` style (which needs you to drop your own `data/avatar/model.glb`), these render immediately and play real skeletal animations mapped to the CoS agent state — `idle` while thinking, `walk` while coding, `sit` while sleeping, `sprint` while ideating, `emote-yes` while reviewing — plus a slow turntable rotation and a speaking head-bob, with clips crossfading on state change. The models are Kenney's [Mini Characters](https://kenney.nl/assets/mini-characters) (Creative Commons Zero), re-exported with the texture embedded and Draco compression disabled so they render offline / over Tailscale without fetching an external decoder from a CDN. Each is ~210 KB and ships in `data.reference/avatar/` (seeded into `data/avatar/` by `npm run setup:data`). The avatar route gained a safe `?variant=<name>` selector (strict `[a-z0-9-]` validation, no path traversal) so multiple bundled characters share one endpoint. Drop any rigged GLB at `data/avatar/<name>.glb` to add your own.

- **[issue-1092] Vocal takes remember their pitch analysis** — when you record a take against a song's notated melody, the take now keeps its color-match grading (percent-in-tune and per-note accuracy) and the tuner trace. Each saved take shows its score in the takes list, and a Review button replays that grading on the staff without re-singing.

- **Public API surface with per-API auth gating (Voice/TTS + sdapi):** PortOS can now serve individual services as HTTP APIs on your tailnet. A new **Settings → API Access** tab exposes per-API toggles — *Expose on the network* (off by default) and *Require auth* (off = passwordless) — so you can keep the whole app behind your PortOS password while leaving the **Voice/TTS** and **Image Gen (sdapi)** APIs open for other machines to call. A deliberately-minimal `POST /api/voice/public/synthesize` (plus `/voices` and `/engines`) turns text into WAV audio with any engine/voice/rate; config and process-control endpoints always stay gated. A single in-tree registry (`server/lib/apiRegistry.js`) drives the auth-gate exemptions, the settings UI, and a native **OpenAPI 3.1 spec** at `GET /api/api-docs/openapi.json` (no swagger dependency — request bodies are generated from the same Zod schemas the routes validate against). The API Access tab shows each API's public base URL and a copy-paste `curl` example, and the Voice settings tab links to it. Defaults are safe: both APIs ship not-exposed and passwordless-once-exposed, with a migration seeding the key for existing installs.

- **Live TUI runs are now viewable AND drivable in the Shell page** — one-shot AI runs that drive a terminal UI (Claude Code, Codex, etc.) — like the manuscript editor's editorial review and pipeline stages — now appear in **Developer Tools → Shell** as fully interactive session tabs (a pulsing bot-icon "live" tab, plus a live-run count in the header). Previously these spawned an invisible PTY you couldn't see; now you can open the tab to watch the run's live output, scroll its recent history (last 50 KB buffered for late attach), **type into it** to answer a question the model is waiting on or steer it mid-run, and **Stop** it if it hangs. So a watched run isn't snapped shut while you're reading or about to answer, the runner pauses its 8-second idle-auto-complete while you have the tab open — the run then ends on natural exit, an explicit Stop, or its configured max-time backstop (unattended pipeline runs keep the snappy idle timing). Tabs are labelled by the run's source (e.g. `pipeline-manuscript-completeness`), don't count toward the shell-session cap, and are skipped by Shell's auto-attach (you opt in by clicking), so the default "give me a shell" behavior is unchanged. The interactive-shell cap was also raised from 5 to 20. (`server/services/shell.js`, `server/lib/tuiPromptRunner.js`, `client/src/pages/Shell.jsx`)

## Fixed

- **Workspace-root validation on git/scaffold routes** — git operations and app scaffolding refuse paths outside allowed workspace roots; file staging uses literal pathspecs (no git glob expansion); `env` removed from the shell command allowlist so process secrets can't land in persisted job output.
- **Route validation + linkable tabs** — jira, loops, CoS, digital-twin, automation-schedule, and history inputs are now Zod-validated; Prompt Manager and Messages triage tabs live in the URL (deep-linkable); previously-swallowed client errors are logged.
- **Helper consolidation** — filename sanitization and MIME maps unified in fileUtils (was 3 copies); GSD/app document saves use atomic writes; client components use the canonical formatters and shared command API instead of local re-implementations.
- **Service extractions** — universe-builder render orchestration, brain bucket cascade/ordering, system folder-open, and reciprocal-rank-fusion moved out of routes into services/lib; the canonPrompt client/server mirror is now enforced by a parity test.
- **Loop fallback crash + durability** — provider fallback no longer crashes loop iterations; loops.json writes are atomic and serialized; cos-runner state races, leaked kill timers, and unguarded close handlers fixed; N+1 issue scans eliminated with an uncapped accessor so >1000-issue installs don't miscount; HF repo-size and Telegram check-in caches bounded.
- **Crash-proofing + UI hygiene** — voice socket handlers can't take down the server on settings-read failures; hung deploy scripts time out instead of wedging the deploy lock; buttons get explicit types, list keys stabilized, theme/notification fetches gain abort/unmount guards.

- **A watched TUI run no longer false-times-out after finishing its work** — companion fix to *Live TUI runs are now viewable AND drivable in the Shell page* (above). One-shot AI runs that drive a terminal UI (Claude Code, Codex, etc.) write their completed answer to a response file and then return to an interactive prompt **without exiting**. The runner only auto-completed such a run via output-idle — which is paused while you watch it in the Shell page (so it isn't snapped shut mid-read). With the process never exiting and idle paused, a watched run had no terminus but the 10-minute hard timeout — at which point it was marked **failed**, its already-written result was **discarded**, and a useless fallback retry ran in its place. The runner now watches for the response file as the model's authoritative "done" signal and completes the run the moment it's written — even while you're watching — since once the file exists the task is genuinely finished. A second safety net salvages the file on hard timeout: if the result was written but completion stalled for any other reason, the run finishes as **success** with that result instead of throwing it away and falling back. So a long editorial pass you keep open in the Shell tab now lands its findings when the model finishes, instead of silently dying at 10 minutes. (`server/lib/tuiPromptRunner.js`)

- **Catalog user-type sync now converges instead of re-applying every cycle** — federated catalog *user-defined types* used an at-or-newer (`>=`) last-write-wins comparison, so a peer re-sending a type you already held at the same timestamp was re-adopted, re-written, and re-counted as a change on every sync cycle. The logs showed a constant non-zero count (e.g. `Synced with <peer>: 3 catalog changes`) once a minute forever, and the slice was rewritten needlessly each pass. The merge now uses strict-newer (`>`), matching the SQL LWW guard every other catalog kind already uses (`EXCLUDED.updated_at > …updated_at`): an equal-clock echo is a skip, so the merge is idempotent and the cycle settles to zero changes. (`server/services/catalogSync.js`)

- **[issue-1080] Notes synced from other machines become searchable promptly** — a Brain note, daily-log entry, person, project, or idea created or edited on one of your machines is now re-indexed into semantic memory on the others as soon as it syncs, instead of waiting for a full manual re-sync. Edits and deletes that arrive from a peer also refresh or retire the matching memory entry, so search no longer surfaces stale copies of records that changed elsewhere. A new **Refresh embeddings** button on the Brain graph re-indexes everything in one pass to heal anything that drifted before this fix.
