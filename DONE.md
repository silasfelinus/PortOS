- **[per-season-and-per-field-locks-extend-arc-lock] Per-season editorial locks.** Extends the existing series-level arc lock to a per-volume editorial freeze. New `season.locked: boolean` field on each season record (`server/lib/storyArc.js#sanitizeSeason`, default `false`, strict-true coercion so a `locked: 'yes'` typo can't accidentally freeze a volume), opt-in through the existing `seasonSchema.passthrough()` in `server/routes/pipeline.js` (explicit `.locked z.boolean().optional()` per the CLAUDE.md "all inputs validated" convention). Three rewriter gates fire on `locked === true`: (1) `arcPlanner.generateSeasonEpisodes` refuses before any LLM call — same `ERR_VALIDATION` code as the existing arc-lock for symmetry; (2) `issuesSvc.bulkReassignSeason` refuses to move issues OUT of or INTO a locked volume (new `ERR_SEASON_LOCKED` code, mapped to HTTP 409) — checks the latest season state inside `queueIssueWrite` even though the source-of-truth is on the *series* file (which has its own queue), since the per-season-lock toggle isn't a hot path and a stale read here would let a single race window leak in; (3) `seasonsSvc.deleteSeason` refuses (new `ERR_LOCKED` code, also 409).
